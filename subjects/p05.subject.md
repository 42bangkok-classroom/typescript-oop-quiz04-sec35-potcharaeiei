# โจทย์ที่ 5: สร้าง User ใหม่ด้วย DTO และ Validation

**Path:** `POST /users`

**Body:** JSON object ที่มีข้อมูล user ใหม่

**ผลลัพธ์:** สร้าง user ใหม่และบันทึกลงไฟล์ `data/users.json` พร้อม return ข้อมูล user ที่สร้าง

**ไฟล์ที่ต้องสร้าง:**

- `src/user/dto/create-user.dto.ts` (class `CreateUserDto` พร้อม class-validator decorators)

**ไฟล์ที่ต้องแก้ไข (ไฟล์ที่มีอยู่แล้วจากข้อ 2-4):**

- `src/user/user.service.ts` (เพิ่ม method `create(dto: CreateUserDto)` เพื่อสร้าง user ใหม่และบันทึกลงไฟล์)
- `src/user/user.controller.ts` (เพิ่ม endpoint `POST /users` พร้อม ValidationPipe)

**ตัวอย่างการเรียก API:**

1. สร้าง user สำเร็จ:

```bash
curl -X POST http://localhost:3000/users \
  -H "Content-Type: application/json" \
  -d '{
    "firstName": "ทดสอบ",
    "lastName": "ระบบ",
    "email": "test@example.com",
    "username": "test_user"
  }'
```

```json
{
  "id": "11",
  "firstName": "ทดสอบ",
  "lastName": "ระบบ",
  "email": "test@example.com",
  "username": "test_user"
}
```

2. Validation error (firstName หรือ lastName ว่าง):

```bash
curl -X POST http://localhost:3000/users \
  -H "Content-Type: application/json" \
  -d '{
    "firstName": "",
    "lastName": "ระบบ",
    "email": "test@example.com",
    "username": "test_user"
  }'
```

```json
{
  "statusCode": 400,
  "message": ["firstName should not be empty"],
  "error": "Bad Request"
}
```

**เงื่อนไข Validation:**

1. **firstName** (required):
   - ต้องไม่ว่าง

2. **lastName** (required):
   - ต้องไม่ว่าง

**หมายเหตุ:**

- ต้องใช้ `class-validator` สำหรับ validation
- เมื่อสร้างสำเร็จ ให้ generate `id` ใหม่ (เป็น string ตัวเลขถัดไป)
- ถ้า validation fail ให้ throw `BadRequestException` (400 status code)
- ต้องเขียนข้อมูลใหม่ลงไฟล์ `data/users.json` (อ่านไฟล์เดิม, เพิ่ม record ใหม่, เขียนกลับ)
